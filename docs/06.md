## 一、什麼是 Literal 型別

- Literal 型別代表一個固定的值本身。
- 它比 `string`、`number`、`boolean` 這類一般型別更精準。
- 例如某個值只能是指定字串，而不是任意字串。

## 二、為什麼需要 Literal 型別

- 很多情境不是接受任意值，而是只能接受固定選項。
- 使用 Literal 型別可以讓型別限制更明確。

## 三、Literal 型別的常見種類

- 字串 Literal 最常見，實務上使用最多。
- Literal 型別常會搭配聯集一起使用。
- 數字 Literal 可用在固定狀態碼、步驟值等情境。
- 布林 Literal 也存在，但實務上較少單獨使用。

## 四、什麼時候用 Literal，什麼時候用一般型別

- 如果資料是自由輸入內容，用一般型別即可。
- 如果資料是固定選項，用 Literal 型別更合適。
- 判斷重點是：這個值有沒有明確邊界與限制。

## 總結

- Literal 型別用來描述固定值，不是描述某一大類型別。
- 它的核心價值是讓型別更窄、更精準。
- 適合用在「只能是特定幾個值」的場景。

- 什麼情況應該使用一般型別（如 `string`），而不是 Literal 型別？

## 課後行動

- 檢視目前專案中使用 `string` 的地方，找出實際上是固定選項的欄位，改為 Literal 型別（例如狀態、模式、類型）。
- Code Review 時，可以多注意是否有應該用 Literal 卻用了 `string` 的情況。
