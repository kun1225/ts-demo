## 一、任意型別

- `any` 代表可以是任何型別。
- 一旦使用 `any`，TypeScript 幾乎不再做型別檢查。
- 可以自由賦值、呼叫屬性或方法，但也失去型別保護。
- `any` 常出現在型別尚未確定的過渡階段。

## 二、`any` 的風險

- 會讓 TypeScript 退回接近 JavaScript 的狀態。
- 編譯時可能不報錯，但執行時容易出現錯誤。
- `any` 會污染後續型別，讓錯誤一路往下傳。
- 函式參數若濫用 `any`，特別容易造成難追蹤的 bug。
- 失去可讀性，不確定函式的細節，容易出 bug

## 三、什麼是 `unknown`

- `unknown` 也能接收任何型別。
- 和 `any` 不同，`unknown` 在未確認型別前不能直接操作。
- 使用前必須先做型別檢查。
- 可以把它理解成「安全版的 `any`」。

## 總結

- 實務上優先用 `unknown`，避免隨手使用 `any`。
