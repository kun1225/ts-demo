## 原始資料型別

TypeScript 常見的原始資料型別有：

- `boolean`
- `number`
- `string`
- `null`
- `undefined`
- `symbol`
- `bigint`

## 如何指定型別

在變數後面加上 `: 型別` 即可：

```ts
let count: number = 1;
let message: string = "hello";
```

也可以先宣告、之後再賦值：

```ts
let count: number;
```

## 布林值（boolean）

- 只會有 `true` 或 `false`
- 常用在條件判斷、狀態切換、權限控制
- 命名常見寫法：`is...`、`has...`

提醒：

- 使用 `boolean`，不要用 `new Boolean()`
- `Boolean(value)` 可以用來做真假值轉換

## 數值（number）

- TypeScript 裡所有數字都屬於 `number`
- 包含整數、小數、負數
- 也支援二進位、八進位、十六進位寫法

提醒：

- `NaN` 代表不是有效數字
- `Infinity` 代表無限大

## 字串（string）

- 用來表示文字資料
- 可用單引號、雙引號或反引號
- 實務上建議專案內保持一致風格

範本字串：

- 使用反引號 `` ` ``
- 可搭配 `${}` 插入變數或運算結果

## Null 與 Undefined

兩者都表示「沒有值」，但意義不同：

- `undefined`：已宣告，但還沒賦值
- `null`：刻意設定為空值

實務建議：

- `undefined` 表示「尚未設定」
- `null` 表示「有這個欄位，但目前刻意留空」
- 團隊要統一空值規則

嚴格模式提醒：

- 開啟 `strictNullChecks` 後，`null` / `undefined` 不能直接指派給一般型別
- 若欄位允許空值，要明確寫成聯合型別，例如：`string | null`

## 空值（void）

- `void` 代表「函式不回傳值」
- 常見於事件處理、紀錄 log、只做副作用的函式

重點：

- `void` 主要用在函式回傳型別
- 幾乎不會拿來宣告一般變數

### `void` vs `undefined` vs `null`

- `void` → 函式的空：這個函式不會回傳任何東西。
- `undefined` → 系統的空：這個值還沒被設定。
- `null` → 人為的空

## 本章重點

- `boolean`：真假值
- `number`：所有數字
- `string`：文字資料
- `undefined`：尚未賦值
- `null`：刻意留空
- `void`：函式沒有回傳值
