## 一、為什麼需要 Utility Types？

- 從既有型別快速加工出新型別。
- 避免重複維護多份相似型別。
- 讓型別能隨原始結構同步演進。

## 二、結構操作類

- `Partial<T>`：全部欄位改 optional。
- `Required<T>`：全部欄位改必填。
- `Readonly<T>`：全部欄位改唯讀。
- `Pick<T, K>`：只挑需要的欄位。
- `Omit<T, K>`：排除特定欄位。
- `Record<K, V>`：建立 key-value 結構。

## 三、Union 操作類

- `Extract<T, U>`：取出符合條件的 union 成員。
- `Exclude<T, U>`：排除符合條件的 union 成員。
- `NonNullable<T>`：移除 `null` 與 `undefined`。

## 四、函式與非同步類

- `ReturnType<T>`：取得函式回傳型別。
- `Parameters<T>`：取得函式參數型別。
- `Awaited<T>`：解開 Promise 內部型別。

## 五、Utility Types 組合使用

- 可多層組合，對準實際需求。
- 常見例子是 `Partial<Pick<T, K>>`。

## 六、其他補充工具

- `ConstructorParameters<T>`：取 constructor 參數型別。
- `InstanceType<T>`：取 class 實例型別。
- `NoInfer<T>`：阻止某些位置參與推斷。
- `ThisType<T>`：控制物件字面值中的 `this` 型別。

## 總結

- Utility Types 是 TS 實務非常高頻的工具。
- 先熟 `Partial`、`Pick`、`Omit`、`Record`，就能處理大部分場景。
