## 一、函式參數與回傳型別

- TypeScript 可以替函式參數與回傳值加上型別。
- 這樣能在呼叫函式時提早檢查錯誤。
- 一般函式與 Arrow Function 都能標註型別。

## 二、函式型別

- 函式本身也可以被當成一種型別。
- 可以先定義函式型別，再套用到多個函式上。
- 適合用在 callback、事件處理器、可重複使用的函式介面。

## 三、Optional 與 Default 參數

- Optional 參數表示該參數可以省略。
- Default 參數表示沒傳值時會使用預設值。
- 兩者都能和型別標註一起使用。

## 四、用 Literal 型別限制函式參數

- 如果函式只接受固定選項，適合用 Literal 型別限制。
- 常見於狀態、尺寸、排序方向、語系代碼等參數。

## 五、Rest 參數

- Rest 參數用來接收不固定數量的參數。
- TypeScript 仍然可以替這些參數做型別約束。
- 好處是，當參數數量不固定時，我們仍然可以讓 TypeScript 清楚知道每個參數應該是什麼型別。

## 六、`void` 與 `never`

- `void` 表示函式沒有回傳值。
- `never` 表示函式不會正常結束。
- `never` 常用於丟錯、無限迴圈與完整性檢查。

## 七、函式過載

- 函式過載讓同一個函式名稱支援多種呼叫方式。
- 可以建立「不同輸入對應不同輸出」的精確關係。
- 如果只是單純聯集型別，未必需要用到過載。

## 課後行動

- 在團隊中處理狀態或聯合型別（例如 API 狀態、UI 狀態）時，嘗試加入 `never` 檢查，確保所有情況都被處理。
