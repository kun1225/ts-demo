## 一、為什麼你需要泛型？

- 避免為不同型別重複撰寫相同邏輯。
- 取代 `any`，保留彈性的同時，維持精準型別。

## 二、泛型的基本語法

- 常見寫法是 `<T>`。
- `T` 只是命名慣例，不是關鍵字。
- 可依語意改成 `U`、`K`、`Item` 等名稱。
- 習慣上：
  - `T`：一般資料型別
  - `K`：物件 key
  - `V`：物件 value
  - `U`：另一個型別
- 可以根據情況調整泛型名稱

## 三、TypeScript 怎麼推斷泛型？

- 多數情況下，TS 會根據參數自動推斷泛型。
- 需要時也能手動指定型別參數。

## 四、多個泛型參數怎麼用？

- 常見組合有 `T`、`K`、`V`、`U`。
- 多泛型是蠻常見的用法，像是上面描述輸入輸出轉換的情境就很實用。

## 五、泛型也能用在 `type` 與 `interface`

- 適合描述「結構固定，但裡面的資料型別不固定」的型別。
- 也可以用泛型組合出一些好用型別工具

## 六、泛型約束：`extends`

- 泛型彈性高，但彈性高，TS 越不知道你打算怎麼使用這個型別。
- 用 `extends` 限制泛型必須符合某種結構。

## 七、泛型搭配 `keyof`

- 從物件身上拿到「有哪些合法欄位可以用」。
- 可限制 key 必須來自物件本身。

## 八、泛型的預設型別參數

- 可以先提供預設值，降低使用成本。
- 適合函式庫、工具型別、元件 API 設計。

## 總結

- 泛型的核心是：不先把型別寫死，而是保留一個「之後再決定」的型別參數。

## 課程驗收

- 為什麼需要泛型（Generic）？
- `<T>` 在泛型中代表什麼？
- `extends` 在泛型中扮演什麼角色？它如何限制型別範圍？
- `keyof` 搭配泛型的用途是什麼？請說明 `K extends keyof T` 的意義。

## 課後行動

- 找出專案中重複邏輯但型別不同的函式，改寫為泛型函式（如 `getFirst<T>`）。
- 如果團隊沒有定義 API 回傳格式，可以主動統一改為泛型型別（如 `ApiResponse<T>`），提升可重用性。
- 檢查有沒有整理成泛型的內部工具，例如`Nullable<T>`、`ApiResponse<T>`
